sass稍稍进阶

其实这篇文章也写了很久了，只不过在之前的hexo主题，后来换主题的时候，折腾了各种情况，就没有了。。。现在补发一下。主要说一下sass的稍稍进阶的东西，感觉把这些都熟练运用，感觉sass在平常的使用也是十分的高效的。

让一切变成字符串 #{}

我通常情况下，应该说在看这篇博文之前，我只知道，sass的变量只能够当属性值来使用，但是如果要把变量当成属性的话，可以吗，嗯，首先按常规出牌：

$attr: color;
p {
    $attr: red;
}

额~上面那片博文说会报错，但是我这边编译就不会报错，但是什么都没发生，我这边的的ruby版本是：2.2.2，sass版本是：3.4.15；这样不可以，嗯~这个时候就有大神出来了#{};这个关键字是用于把我们声明的变量变成字符串形式，然后上面的代码可以改成：

// 可以横着用
$attr: color;
p {
    #{$attr}: red;
}
// 也可以竖着用
$className: box;
div.#{$className} {
    width: 100px;
}

ES6的一些特性(三)

这一篇主要介绍ES6的Class,模块，生成器等等。文章都给卡住了，因为在理解生成器那里浪费了挺多时间，而且理解还不是很好，今天也是憋了一股劲，才把文章写了。。。

Class

ES6之前，并没有类的定义，但是可以通过prototype来模拟出类，但是继承是通过原型继承，而不是类继承,ES6引入了类的概念，作为对象的模板。通过class关键字可以定义类

class Point {
    constuctor(x, y) {
        this.x = x;
        this.y = y;
    }
    toString() {
        return '(' + this.x + ', ' this.y + ')';
    }
}

constructor方法是类的默认方法，通过new命令生成对象实例时，自动调用该方法。一个类必须有constructor方法，如果没有显式定义，一个空的constructor方法会被默认添加。

// 生成实例对象
var point = new Point(1,3);

需要注意的是，类里面的方法不需要使用function关键字来定义，每个方法之间也不需要用逗号分开，如果使用了逗号，反而会报错

字符串的扩展

ES6加强了对Unicode的支持，扩展了API;以前javascript的每个字符是固定两个字节的。在ES6，对于那些需要4个字符存储的字符(Unicode编号大于0xFFFF的字符)，Javascript会认为它们是两个字符。对正则的支持也增多了。

但是觉得最厉害的是，字符串增加了模版字符串，for example

// 普通字符串
`In JavaScript is a good.`
// 多行字符串
`In ES5 JavaScript this is
not legal`

let 和 const

let命令

在ES6之前，声明变量使用 var 关键字，使用var声明的变量会“变量提升”，因此，变量的作用域是函数作用域而不是类似java那样的块级作用域。而使用ES6定义变量关键字let，可以使得javascript增加了块级作用域。

function f1() {
    let n = 5;
    if(true) {
        let n = 10;
    }
    console.log(n); // 5
}
