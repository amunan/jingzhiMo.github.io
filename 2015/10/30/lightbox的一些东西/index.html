lightbox的一些东西

  • Javascript
  • css

之前这几天，感觉突然好想不知道做什么好，node好像不知道看什么，mongodb入门了那么一点点(做毕业设计),，然后，好像也有段时间没有敲代码了，感觉要写点东西什么的来玩玩。

其实在很久之前(一两个月吧)，就很想写个插件什么的，然后方便后面的快速开发；前几天也去慕课网逛逛，咦！发现一个教程是关于Lightbox的插件，然后？就没然后了。看了一下，顺便了解一下大概的思路，就开始编写了…lightbox是什么？嗯，刚开始我也比它这个名字吓到了，好像很高大上的样子，原来发现是这样子的：


嗯，没错，就是平常用来查看大图的东西，可以上/下一张，显示图片描述；然后抽出来，变成一个插件，方便使用。在编写的过程，也遇到一点问题：…

如何封装

这个问题也是困扰了一段时间，因为不知道怎样去封装，才算是“及格”，后来在慕课网的教学视频里面看到的，是这样子的：

(function() {

    // ...

    // 暴露方法给外面调用
    window.xxx = xxx;

})();

把自己的代码放到一个匿名的，自执行的回调函数里面，然后再通过window来暴露方法给外面调用；这样子的话，就可以把里面的变量使用闭包“藏起来”，只通过向外面暴露的方法，可以让外部来进行一些操作。

使用单例模式

一般情况，一个页面就只有一个lightox来显示图片，无论页面有多少个小图，只要把大图的一些参数传递，就使用这个lightbox来显示，所以我这里选用了单例模式；把init的方法暴露给外面，使用者哪怕多次调用，内存当中，也只有一个实例对象，这样的话，一定程度上可以提高性能。

在切换的时候，由于两张图片的大小是不同的，所以要使用动画进行改变宽度高度；但是，暂时来说，不是动画的问题，而是，怎么获取到图片的宽度高度？难道直接加载图片，然后改变img的src？这样可以撑开外层容器的大小，但是怎么获取w/h?翻了以前的笔记，看到有这种方法：

var img = new Image();
img.src = '..';
img.onload = function() {

    console.log(img.width);
    console.log(img.height);
};

通过new一个图片的对象，然后放到内存里面，通过onload的回调函数，即可获取到宽度高度

纠结！用纯js还是jq

其实在写插件的时候，打算是用纯js来写的，这样的话，就不用再依赖一个工具…嗯，再获取元素那里还好，没问题，再事件委托那里，还好，也顶得住，但是后来发现，需要使用动画，动态变化图片容器的w/h的时候；马上醉了，好吧，还是用个jquery吧。顺便有空去看看jquery的animate动画是怎么实现的。

垂直居中

这次在显示垂直居中的时候，也遇到一点问题，所以才有这篇文章，因为刚开始使用这种：

#out {
    position: relative/absolute/fixed; // 这三个，随便一个都可以
}
#in {
    position: absolute;
    top: 0px;
    right: 0px;
    bottom: 0px;
    left: 0px;
    margin: auto;
}

PS: 具体这种方法可以参考垂直居中文章的链接

但是发现容器一直占满最大宽度高度，不能够由里面的元素撑开，后来就算了这种方法：

#out {
    position: relative/absolute/fixed; // 这三个，随便一个都可以
}
#in {
    position: absolute;
    left: 50%;
    top: 50%;
    width: 100px;
    height: 100px;
    transform: translate(-50%, -50%);
}

虽然所说IE兼容不好，但是…算了，兼容迟点再弄一下吧；放弃IE…哈哈
