实现垂直居中的一些方法

  • css

最近几天在写一个关于弹出显示大图的lightbox插件，有兴趣的同学可以点进去看看，哈。不过编写这个插件的时候，也遇到过一个垂直居中的方法，也卡了一阵子，后来翻翻以前的笔记，顺便总结一下，各种实现垂直居中的方法，还有对应的优缺点。嗯，还有，因为使用javascript来写的话，实现起来就比较容易，所以下面的方法都是使用css代码来实现，来来来，瞄一下：

下面的dom结构，在下面方法的那些例子都会用到，所以，嗯哼，继续：

<div id="out">
    <div id="in"></div>
</div>

使用绝对定位与margin结合的方法实现

css代码如下：

#out {
    position: relative/absolute/fixed; // 这三个，随便一个都可以
}
#in {
    position: absolute;
    left: 50%;
    top: 50%;
    width: 100px;
    height: 100px;
    margin-left: -50px;
    margin-top: -50px;
}

优点： 兼容性好，容易实现
缺点：需要知道里面元素的宽度高度，

使用绝对定位与transform的方法实现

css 代码如下：

#out {
    position: relative/absolute/fixed; // 这三个，随便一个都可以
}
#in {
    position: absolute;
    left: 50%;
    top: 50%;
    width: 100px;
    height: 100px;
    transform: translate(-50%, -50%);
}

优点：容易实现，不需要知道里面元素的宽度高度
缺点：使用CSS3的一些特性，旧版本浏览器，如IE8及其以下都不可以

使用line-height和vertical-align实现

css代码如下：

#out {
    height: 500px;
    line-height: 500px;
    vertical-align: middle;
    text-align: center;
}
#in {
    display: inline-block;
    vertical-align: middle;
}

优点：不需要知道里面元素的宽高，兼容性也不错
缺点：需要知道容器元素的高度,但是宽度可以不知道

这个方法比较神奇，我也不知道怎么说，看代码吧

#out {
    position: relative/absolute/fixed; // 这三个，随便一个都可以
}
#in {
    position: absolute;
    top: 0px;
    right: 0px;
    bottom: 0px;
    left: 0px;
    margin: auto;
}

优点：兼容性好，容易实现，里面，外面元素的高度宽度都不需要知道
缺点：一开始，我以为这个方法最厉害，没缺点；直到我写个lightbox的插件的时候发现…#in元素不能由它的子元素撑开，而是一直占到#out的大小的最大；这个有时候很不好，因为有时候想让#in元素让里面的撑开，但是#in会一直占到最大的宽度高度

对的，最厉害的，往往是最后出场的

使用弹性布局

#out {
    display: flex;
}
#in {
    margin: auto;
}

优点： 代码粗暴，呵呵
缺点：需要厉害一点点的浏览器，要支持弹性布局

最后

顺便说一下为什么很多定位都是:position: relative/absolute/fixed;；之前的我，是认为，如果定位要position:absolute的话，就需要找到离它最近的父节点，该父节点还需要是position:relative；但是后来发现，只需要，父节点定位非静态(position:static)，都可以；而静态定位的话，就是默认值，所以，只要不是默认值即可。嗯~
