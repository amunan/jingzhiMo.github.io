sass稍稍进阶

  • css
  • sass

其实这篇文章也写了很久了，只不过在之前的hexo主题，后来换主题的时候，折腾了各种情况，就没有了。。。现在补发一下。主要说一下sass的稍稍进阶的东西，感觉把这些都熟练运用，感觉sass在平常的使用也是十分的高效的。

让一切变成字符串 #{}

我通常情况下，应该说在看这篇博文之前，我只知道，sass的变量只能够当属性值来使用，但是如果要把变量当成属性的话，可以吗，嗯，首先按常规出牌：

$attr: color;
p {
    $attr: red;
}

额~上面那片博文说会报错，但是我这边编译就不会报错，但是什么都没发生，我这边的的ruby版本是：2.2.2，sass版本是：3.4.15；这样不可以，嗯~这个时候就有大神出来了#{};这个关键字是用于把我们声明的变量变成字符串形式，然后上面的代码可以改成：

// 可以横着用
$attr: color;
p {
    #{$attr}: red;
}
// 也可以竖着用
$className: box;
div.#{$className} {
    width: 100px;
}

这样就可以编译了，编译的结果是：

p {
      color: red;
}
div.box {
      width: 100px;
}

sass中的除法

其实下面也有说到关于sass的四则运算的，但是由于除法的坑有点多，so..除法有很多需要注意的地方，因为除法的运算符 / 在CSS中也有遇到，例如；font的缩写属性 font-size 和 line-height 属性，就需要 / 来分割。所以，在这些包含 / 的CSS属性中对应的位置，是不会参与运算的，除了下面的情况：

  • 如果两个值其中一个或两个存放在变量中，或者由函数返回的值；
  • 如果值被包裹在一对括号里面
  • 如果值作为另一个表达式的一部分

下面的是例子：

$width: 1000px;
p {
    font: 10px/8px;             // 纯 CSS 不会运算$width: 1000px;
    width: $width/2;            // 使用变量，执行运算
    width: round(1.5)/2;        // 使用函数返回值，执行运算
    height: (500px/2);          // 使用括号包裹，执行运算
    margin-left: 5px + 8px/2px; // 用了加法，作为表达式的一部分，执行运算
}

编译后输出：

p {
    font: 10px/8px;
    width: 500px;
    height: 250px;
    margin-left: 9px; 
}

Sass 运算注意事项

建议运算符之间空开一个空格。

注意运算单位

注意10px 也被看成数值型，可以进行数值运算，但是对于单位是有要求的。单位也会参加运算。在运算中，两个操作数单位可以一致，也可以一个数没有单位，而对于不同的运算操作符，单位运算结果也不同，一般说测试 结果如下：

  • 加法：都没有单位输出纯数字；一方有单位，则结果输出该单位；两方相同单位，结果输出该单位；双方单位不同，报错。
  • 减法：类似加法。
  • 乘法：两方相同单位，报错；一方有单位，结果输出该单位；两方都无单位，输出无单位。
  • 除法：两方相同单位，结果无单位；都没有单位，结果无单位；一方有单位另一方无单位，报错。居然除法都没有单位输出。。。所以我们需要在后面加上 0px 这样带单位的运算，结果才会有单位(10px / 5px + 0px)

@if、@else if、@else控制命令

$color-theme: orange;
@if $color-theme == pink {
    background-color: pink;
} @else if $color-theme == orange {
    background-color: orange;
} @else {
    background-color: black;    
}

感觉配合@function 一齐使用会好一点

@for 循环

这个循环比较适用于简单的循环，只能使用数字，且每次的步长为1

@for $i from 1 through 4 {
    .buy_#{$i} {
        background-image: url(buy_#{$i}.png);
    }
}

生成的代码：

.buy_1 {
    background-image: url(buy_1.png);
}

.buy_2 {
    background-image: url(buy_2.png);
}

.buy_3 {
    background-image: url(buy_3.png);
}

.buy_4 {
    background-image: url(buy_4.png);
}

这个for 循环是从1到4，其实 through 的关键字可以换成 to，不过有一点点差别；例如：
@for $i from 1 to 4

这个循环其实输出1到3，并没有到4

@while 循环

while循环跟for循环差不多，一般都是用数字来做循环的，但是步长是可以改变，要等到判断条件是false的时候才会退出循环

$i: 6;
@while $i > 0 {
    .item#{$i} {
        background-image: url(img-#{$i}.png);
    }
    $i: $i - 2 // 这里修改步长
}

输出：

.item6 {
  background-image: url(img-6.png);
}

.item4 {
  background-image: url(img-4.png);
}

.item2 {
  background-image: url(img-2.png);
}

@each 循环

这个循环比@for，@while循环都要灵活，实际生产中，使用这个应该比较多

第一种用法，最简单粗暴的用法

例如现在我们要把几个icon的样式输出

@each $icon in up,down,left,right {
    .#{$icon} {
        background-image: url(img-#{$icon}.png);
    }
}

输出：

.up {
  background-image: url(img-up.png);
}

.down {
  background-image: url(img-down.png);
}

.left {
  background-image: url(img-left.png);
}

.right {
  background-image: url(img-right.png);
}

第二种用法，使用变量把元素存储起来

但是上面把要循环的icon名称都写到each循环里面，好像不太好，所以可以用变量存储起来，然后遍历变量： $list: up,down,left,right，输出与上面那个也是一样的

@each $icon in $list {
    .#{$icon} {
        background-image: url(img-#{$icon}.png);
    }
}

第三种用法，获取下标

上面两个用法都相对简单，来试下相对复杂那么一点点的东西：如果我们需要知道当前遍历到的元素的下边是多少

@each $icon in $list {
    $idx: index($list, $icon);
    .box {
        &:nth-child(#{$idx}) {
            background-image: url(img-#{$icon}.png);
        }
    }
}

输出：

.box:nth-child(1) {
  background-image: url(img-up.png);
}

.box:nth-child(2) {
  background-image: url(img-down.png);
}

.box:nth-child(3) {
  background-image: url(img-left.png);
}

.box:nth-child(4) {
  background-image: url(img-right.png);
}

第四种用法，遍历元素组

@each $cls, $color, $display in (cls1, #ccc, block),
                                (cls2, #ddd, inline-block),
                                (cls3, #eee, inline) {
    .#{$cls} {
        color: $color;
        display: $display;
    }
}

输出：

.cls1 {
  color: #ccc;
  display: block;
}

.cls2 {
  color: #ddd;
  display: inline-block;
}

.cls3 {
  color: #eee;
  display: inline;
}

简直炫酷鸟

第五种用法，循环映射型map

@each $header, $size in (h1: 2em, h2: 1.5em, h3: 1.2em) {
    #{$header} {
        font-size: $size; 
    }
}

输出：

h1 {
  font-size: 2em;
}

h2 {
  font-size: 1.5em;
}

h3 {
  font-size: 1.2em;
}

@function

这个功能了解不是很深入，不过估计（哈哈）是跟平常写函数差不多，但是要遵循sass的语法，来一碟代码：

@function flow-options($flow: left) {
    @if $flow == right {
        @return right;
    } @else {
        @return left;
    }
}
// 调用函数
$flow: flow-options(right);
.fn {
    float: flow-options(left);
}

定义是用 @function 来定义，然后就…差不多了，应该一看就看懂了吧，然后调用，也挺简单的。看一下sass中一些函数的语法

media query

响应式必备啊，来一发最直接的代码，原生css(233)：

.style {
    width: 100%;
}
@media only screen and (min-width: 320px) {
    .style {
        width: 80%;
    }
}
@media only screen and (min-width: 480px) {
    .style {
        width: 60%;
    }
}
@media only screen and (min-width: 720px) {
    .style {
        width: 40%;
    }
}
// ...

很明显，如果元素多了，简直是，写到。。不想写了，因此SASS又出现了：

@mixin MQ($size) {

    @if $size == XS { // 320以下
        @media screen and (max-width: $XS) {
            @content;
        }
    }
    @else if $size == S { // 321-480
        @media screen and (min-width: $XS + 1) and (max-width: $S) {
            @content;
        }
    }
    @else if $size == M { // 481-720
        @media screen and (min-width: $S + 1) and (max-width: $M) {
            @content;
        }
    }
    @else if $size == L { // 721-960
        @media screen and (min-width: $M + 1) and (max-width: $L) {
            @content;
        }
    }
    @else if $size == XL { // 721-960
        @media screen and (min-width: $L + 1){
            @content;
        }
    }
}

然后调用：

#wrap {
    display: inline-block;
    color: $grey;
    @include MQ(XS) {
        width: 100%;
    }
    @include MQ(S) {
        width: 100%;
    }
    @include MQ(M) {
        width: 80%;
    }
    @include MQ(L) {
        width: 60%;
    }
    @include MQ(L) {
        width: 40%;
    }
}

输出：

@media screen and (max-width: 320px) {
  #wrap {
    width: 100%;
  }
}
@media screen and (min-width: 321px) and (max-width: 480px) {
