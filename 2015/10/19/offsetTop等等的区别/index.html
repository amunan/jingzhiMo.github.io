offsetXXX,scrollXXX,clientXXX等等的区别

  • Javascript

昨晚，有个朋友问了一段代码，是有关获取具体元素的位置的，里面主要涉及两个模糊的东西，一个是offsetTop,另外一个是offsetParent，然后那个函数的主要目的是获取元素到文档左上角的top和left的大小。因为之前我也在这里纠结了很久，因为除了两个概念，还有其他的一些概念；顺便翻了一下权威指南，看到主要有三个系列：offset, client, scroll;这三个系列，这篇文章就明确一下这些概念…

如果分三个系列来写的话，可能对比性不太明显，所以我就把width,height的归一类，top,left的归一类。

width,height

在看这一类的区别的时候，先看一下代码，这些代码会在这一类中，反复使用，但是可能会对不同部分进行注释，以看到不同的效果

<!-- html结构 -->
<div id="out">
    <!--
    <div id="in_1">
    </div>
    <div id="in_2">
    </div>
    -->
</div>

/*css 样式*/
#out {
    width: 100px;
    height: 100px;
    padding: 5px 20px;
    margin: 5px 10px;
    border: 2px solid;
    /*overflow: scroll;*/
}

先看一下定义：

  • offetWidth的定义：元素的offsetWidth 是一个只读属性，返回一个元素的布局宽度。offsetWidth是测量元素的borders、水平线上的padding、垂直线上的scroolbar（如果存在滚的话）、以及css设置的宽度的值。

  • clientWidth的定义：元素的clientWidth的定义属性表示元素的内部宽度，以像素计。该属性包括内边距，但不包括垂直滚动条（如果有的话）、边框和外边距。

  • scrollWidth的定义：元素的scrollWidth只读属性以px为单位返回元素的内容区域宽度或元素的本身的宽度中更大的那个值。若元素的宽度大于其内容的区域（例如，元素存在滚动条时）, scrollWidth的值要大于clientWidth。

是不是有一点晕了= =！

offsetWidth

先看一下offsetWidth，这个包含了元素的宽，内边距，还有边框的大小；这个应该很好理解，所以在上面的结构当中，div#out的offsetWidth的值就是：100 + 20 * 2 + 2 * 2 = 144px;而且测试过，这个属性在IE7+的浏览器和现代浏览器中，都是一致的。

clientWidth

然后再看一下clientWidth，定义说，不包含垂直滚动条和边框，嗯~边框很好理解，但是垂直滚动条又怎么说呢；原来，刚才在审查元素的时候发现，滚动条的位置是在padding上面的，这个有点像平常使用position:absolute定位那样，可以想象就是，滚动条“飘”在padding上面；这个可能有点抽象，拿上面的div#out元素做例子，把#out的样式中的overflow:scroll的注释去掉，元素有100px宽度，左右padding有20px的话,

那么clientWidth = 100 + 20 * 2 - 滚动条宽度;

所以clientWidth的计算公式就是：clientWidth = width + padding-left + padding-right - 滚动条宽度;那么现在问题又有了，滚动条的宽度到底是多少，刚才经过简单的加减法(= =!)发现各个浏览器基本一致，都是17px；但是并不确定，不过连IE7都是，基本也没什么问题了，哈哈

scrollWidth

最后，再看一下scrollWidth，不过觉得这个用scrollHeight来说明，可能会容易一点，把#out元素里面的子元素注释去掉，然后为子元素加上下面的样式：

#in_1 {
    width: 100%;
    height: 150px;
    background-color: #ccc;
}
#in_2 {
    width: 100%;
    height: 100px;
    background-color: #458dcb;
}

很明显，这个时候，外面包裹的div元素不能够包括所有子元素，但是因为设置了超出就显示滚动条，所以，这个时候，垂直滚动条就出现了；如果这个时候获取scrollHeight的话，这个时候的值就是：260px。怎么来的？答案就是：…

#in_1的高度加上#in_2得到高度，再加上10px;嗯，估计有些人已经猜到10px是怎么来的了。就是#out的padding-top和padding-bottom；所以，嗯哼。再看一下定义，“哪个大，返回哪个？”。是的，因为这个时候，容器的实际上高度就是里面元素撑开的，而且比该元素的视口大小要大，所以，这个时候要取里面元素的实际的大小。

但是，如果没有出现滚动条呢？那么这个时候的scrollHeight与clientHeight就是一致的。所以，scrollHeight的大小也是不包括滚动条的大小，和边框的大小。

next…

top,left

好的，我们再来看一下定义：

  • offsetTop的定义：HTMLElement.offsetTop 为只读属性，它返回当前元素相对于其 offsetParent 元素的顶部的距离。
  • scrollTop的定义：指定元素的滚动条的位置
  • clientTop的定义：一个元素顶部边框的宽度（以像素表示）。不包括顶部外边距或内边距。clientTop 是只读的。

offsetTop,offsetLeft

要想比较好的理解这两个概念，需要弄清楚offsetParent，获取一个元素的offsetParent的元素是：ele.offsetParent；但是这个“父元素”是直接上一层的父元素吗？或许再看看定义，会更加清晰一些：

HTMLElement.offsetParent 是一个只读属性，返回一个指向最近的（closest，指包含层级上的最近）包含该元素的定位元素。如果没有定位的元素，则 offsetParent 为最近的 table 元素对象或根元素（标准模式下为 html；quirks 模式下为 body）。当元素的 style.display 设置为 “none” 时，offsetParent 返回 null。offsetParent 很有用，因为 offsetTop 和 offsetLeft 都是相对于其内边距边界的。

定义有点长，我们来选一个关键字，对，没错，就是“定位元素”。那什么才是叫定位元素呢？就是已经定位了的元素…擦；其实是position定义了非static值(position的默认值就是static)的元素，也就是说，只要position只要设置了relative,absolute,fixed着三个值的其中一个，该元素就变成了定位元素了。举个例子：

<body>
    <div id="out">
        <div id="in"></div>
    </div>
    <script>
        var inEle = document.getElementById('in');
        console.log(inEle.offsetParent);
    </script>
</body>

上面的代码当中，在控制台中，打印出来的元素是body，囧。怎么跟定义有出入的。。。额，也不是很清楚，可能根元素就是body吧。大家可以测试一下。

如果我把#out的样式设置一下：position:relative；这样的话，那么打印出来的元素就是，div#out了

嗯，扯了这么长，要回来offsetTop的属性来了，这个时候就容易了，就是当前选中的元素，距离它的offsetParent元素左上角的top高度相差多少。

scrollTop,scrollLeft

这个属性比较简单，就是该元素的滚动条（如果有的话），滑动过的距离。scrollTop就是垂直滚动条滚动的距离，scrollLeft就是水平滚动条滚动的距离。

clientTop,clientLeft

《javascript权威指南》说，这两个属性没什么用；他们返回元素的内边距的外边缘和它的边框的外边缘之间的水平距离，通常这些值就等于左边和上边的边框的宽度。但是如果元素有滚动条，并且浏览器将这些滚动条放置在左侧或顶部（可能不太常见），clientLeft和clientTop也就包含了滚动条的宽度。对于内联元素，这两个值总是0。
