html5和html4的一些区别

上次去面试阿里的时候，被问到关于h5与h4的一些区别，= =！没有怎么总结过，只是记得多少说多少，额~

下面只是简单列几点出来，主要是从 HTML5 程序设计(第2版) 这本书看到的

新的DOCYPE和字符集

H5: 

<!DOCTYPE HTML>

H4: 

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

H4的简直没法记住…都是CTRL+C,CTRL+V…

新增语义化标签

h5增加了很多的标签，其中语义化标签占有一定的比重。虽然语义化标记不会马上感受到什么好处，但是它可以简化HTML页面设计，并且将来搜索引擎在抓取和索引网页的时候，也会利用到这些元素的优势。

语义化的简单理解

  • 用正确的标签做正确的事情！
  • html语义化就是让页面的内容结构化，便于对浏览器、搜索引擎解析；
  • 在没有样式CCS情况下也以一种文档格式显示，并且是容易阅读的;
  • 搜索引擎的爬虫依赖于标记来确定上下文和各个关键字的权重，利于 SEO;
  • 使阅读源代码的人对网站更容易将网站分块，便于阅读维护理解。

canvas API 和 SVG

canvas元素：我们可以通过javascript来自由控制它，在里面加图片、线条以及文字，能够大大丰富了页面的表现，在画布里面绘图，甚至动画

SVG: 另外一种图形功能，可缩放矢量图形，一种二维图形表示语言。

音频(audio)和视频(video)

audio和video元素的出现让HTML5的媒体应用多了新选择，开发人员不必使用插件就能播放音频和视频。对于这两个元素，HTML5规范提供了通用、完整、可脚本化控制的API。

不论音频文件还是视频文件，实际上都是一个容器文件，这点类似压缩了一组文件的ZIP文件。视频文件（视频容器）包含了音频轨道、视频轨道和其他一些元数据。视频播放的时候，音频轨道和视频轨道是绑定在一起的。元数据部分包括了该视频的封面、标题、子标题、字幕的相关信息。主流视频容器支持视频格式如下：

  • .avi
  • .flv
  • .mp4
  • .mkv
  • .ogv

解码器：音频和视频的编码/解码器是一组算法，用来对一段特定视频或视频进行解码和编码，以便音频和视频能够播放。原始的媒体文件体积非常庞大，假如不对其编码，那么构成一段视频和音频的数据可能会十分庞大，以至于在因特网上传播需耗费无法忍受的时间。若没有解码器的话，接收方就不能把编码过的数据重组为原始的媒体数据。编解码器可以读懂特定的容器格式，并且对其中的音频轨道和视频轨道编码。

一些音频解码器：

  • AAC
  • MPEG-3
  • Ogg Vorbis

一些视频编解码器：

  • H.264
  • VP8
  • Ogg Theora

Geolocation API(地理定位API)

HTML5 Geolocation API的使用方法相当接单。请求一个位置信息，如果用户同意，浏览器就会返回位置信息，该位置信息是通过支持HTML5地理定位功能的底层设备（例如笔记本或手机）提供给浏览器的。位置信息由纬度、经度坐标和一些其他元数据组成。

Communication API(跨文档消息通信和XHR Level 2)

由于处于安全性考虑，不同源之间的通信受到了很大的阻碍，有时候两个网站之间需要利用不同的数据，需要进行通信，在h5之前需要用到跨域的手段，但是由于不是原生支持，可能有些实现起来有点麻烦。下面两种是h5提供的跨域方法；

postMessage API

// 假设父窗口里面嵌入了一个子窗口：
<iframe name="msg" id="msg" src="XXX"></iframe>
// 且父窗口与子窗口是不同源的，两个窗口之间通信：
父窗口的地址是：http://localhost:8887/index.html
子窗口的地址是：http://localhost:8886/son.html

对于父窗口来说： 传递信息和接收信息

var sonWin = document.getElementById('msg');
// 发送消息
sonWin.postMessage(JSON.stringify(json), 'http://localhost:8886/son.html');
// 接收消息
window.addEventListener('message', function(ev) {
    console.log'从子窗口接收到的数据是：' + (ev.data);
});

对于子窗口来说：

// 发送消息
window.parent.postMessage(sendData, 'http://localhost:8887');
// 接收消息
window.addEventListener('message',function(ev){
    console.log('从父窗口接收到的数据是：' + ev.data);
},false);

XHR Level 2

XHR leve2提供了CORS(Cross Origin Resource Sharing,跨源资源共享)来实现跨域。跨源HTTP请求包括一个Origin头部，它为服务器提供HTTP请求的源信息。头部由浏览器保护，不能被应用程序代码更改。从本质来讲，它与跨文档消息通信中消息事件的origin属性作用相同。

服务器收到HTTP请求之后，可以查看请求的Origin头部，看是不是在白名单当中，如果是的话，就可以返回对应的response.

WebSockets API

这是html5中最强大的通信功能，它定义了一个全双工通信通道，仅通过web上的一个Socket即可进行通信。

Websocket是web应用程序的传输协议，类似TCP，它提供双向的、按序到达的数据流。和TCP协议一样，高层协议能够在Websocket上运行。作为web的一部分，websocket连接的是URL,而非因特网上的主机和端口。

Forms API

h5的表单增强了好多，，主要分为两类

  • 新的输入型表单空间
  • 新的函数特特性

输入型控件

(冒号前面是input的type类型，冒号后面表示用途)

  • tel: 电话号码
  • email: 电子邮件地址文本框
  • url: 网页的URL
  • Search： 用于搜索引擎，比如在站点顶部显示的搜索框
  • range: 特定范围的数值选择器，典型的显示方式是滑动条
  • number： 只包含数值的字段
  • …

新的表单

(冒号前面是属性，冒号后面是简单解析)

  • placeholder(“message”): 提示信息
  • autocomplete(on, off, unspecified): 用来保护敏感用户数据，避免本地浏览器对它们进行不安全地存储
  • autofocus(): 让控件自动获取焦点
  • spellcheck(true, false): 对输入控件设置，询问浏览器是否应该给出拼写检查结果反馈。(一般那些文本下方出现一条红色虚线)，浏览器默认是开启的

  • list特性和datalist元素
  • min和max(number): 上面介绍的type="ranage"可以是使用两个属性来设置最大和最小值
  • step(number): 对于输入型控件，设置其step特性能够指定输入值递增或递减的粒度。例如range控件
  • required(): 表示必填

拖放

额，拖放事件比较复杂，而我又掌握的不好，所以不太了解。总体来说，就是能够实现html元素的拖动，类似我们在操作系统中的拖动窗口(说了跟没说一样).

Web Worker API

由于javascript是单线程的，因此长时间的计算会阻塞UI线程。该问题解决方案是，H5 Web Worker能够让web应用程序可以充分利用多核CPU带来的优势。将耗时间长的任务分配给Workers执行，可以避免弹出脚本运行缓慢的警告。但是Web Workers不能直接访问Web页面和 DOM API。

Web Storage API

在浏览器里面，在H5出现之前，大多使用cookie来在客户端存储数据，但是数据量不大，一般为4K。而storage提供更大的存储，一般为5M。
