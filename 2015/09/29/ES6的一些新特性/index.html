ES6的一些新特性(一)

  • ES6
  • Javascript

最近几天在看一本ES6的书，了解一下ES6的新的特性，因为篇幅比较多，就挑一些感觉比较厉害的新东西来”复制-粘贴”，但是，还是内容不少，所以分开几篇来吧- - !

如果想在线看完整的文章，可以看一下这里。

let 和 const

let命令

在ES6之前，声明变量使用 var 关键字，使用var声明的变量会“变量提升”，因此，变量的作用域是函数作用域而不是类似java那样的块级作用域。而使用ES6定义变量关键字let，可以使得javascript增加了块级作用域。

function f1() {
    let n = 5;
    if(true) {
        let n = 10;
    }
    console.log(n); // 5
}

如果是使用var定义的话，就会输出10，而不是5。再看一个比较经典的例子，关于javascript作用域问题的：

// 使用var 定义 i 变量
var a = [];
for(var i = 0; i < 10; i++) {
    a[i] = function() {
        console.log(i);
    }
}
a[1](); // 10
// 使用let定义 i 变量
var a = [];
for(let i = 0; i < 10; i++) {
    a[i] = function() {
        console.log(i);
    }
}
a[1](); // 1

由于var会提前声明，第一个for循环，无论执行数组哪一个函数，输出都会是10；而’let’声明的，i的作用域范围只有在花括号(指的是function的花括号)。由于不会声明，所以在定义之前使用该变量，会报错。

function f() {
    console.log(a); // ReferenceError
    let a = 1;
}

还要注意的是，在同一个作用域里面，不能声明同一个变量，否则也会报错。

// error
if(true) {
    var a = 1;
    let a= 2;
}

const命令

·const·关键字用来声明常量的，跟let一样，都是有块级作用域效果，也是不可以重复声明同一个变量。

const PI = 3.1415;
// 如果对常量进行赋值，不会报错，但是不会修改成功
PI = 3;
PI // 3.1415

变量的结构赋值

数组的结构赋值

以前，定义变量的时候，是这样定义的：

var a = 1,
    b = 2,
    c = 3;

现在，我们可以这样定义变量：

var [a, b, c] = [1, 2, 3];

是不是突然亮瞎了..这种赋值方式，叫解构赋值，数组的解构赋值需要按顺序一一对应的，也有下面的几种方式：

var [foo, [[bar], baz]] = [1, [[2], 3]];
// foo: 1; bar: 2; baz: 3
var [,,third] = [1, 2, 3];
// third: 3;
var [head, ...tail] = [1, 2, 3, 4];
// head: 1; tail: [2, 3, 4];
// 三个点(...)也是一种关键字， 在后面可以看到解析

上面的情况都是可以解构成功，也有不成功的，如果解构不成功，声明的对象就等于undefined

// 下面5种情况，foo的值都是 undefined
var [foo] = [];
var [foo] = 1;
var [foo] = 'hello';
var [foo] = false;
var [foo] = NaN;

// 下面两种会报错
var [foo] = undefined;
var [foo] = null;

这是因为解构只能用于数组或者对象。其他原始类似的值都可以转为对应的对象，但是undefined和null不能转化为对象，so,报错了。

解构赋值也可以适用于let和const..

解构的另外一个强大的特性就是可以指定默认值

var [foo = true] = [];
foo // true

看不出强大之处？再往下看…

对象的解构赋值

var {bar, foo} = {foo: '1', bar: '2'};
// bar: "2"
// foo: "1"

从上面能看得出，对象的解构与数组的解构有什么很简单的不同？

嗯.就是顺序可以随便来，只要变量名与对象的属性同名，就可以取到值。但，如果没有对应的对象属性名：

var {c} = {a: "aa", b: "bb"};
// c: undefined

如果变量名与属性名不一致的话，就这样咯：

var {a: c} = {a: "aa", b: "bb"};
// c: "aa"

对象的解构也可以指定默认值

var {x = 3} = {};
// x: 3

对象解构与数组解构混合

var o = {
    p: [
        'hello',
        {
            y: 'world'
        }
    ]
};
var {p: [x, {y}]} = o;
// x: "hello"; y: "world"

用途

说了那么多，不可能只是写少两个var 吧？

  • 交换变量的值
[x, y] = [y, x];
  • 函数返回多个值
function example() {
    return [1, 2, 3];
}
var [a, b, c] = example();
  • 返回一个对象
function example() {
    return {
        foo: 1,
        bar: 2
    };
}
var {foo, bar} = example();
  • 函数参数的定义、默认值，提取JSON对象
function f({x = 3, y = 4, z = 5}) {
    // 如果不传参数，也有默认的参数，不用之前那样写
    // options = options || {};
}
f({x:1, y:2, z: 3});
f({x: 1, y: 2});
// 提取jsondata
var jsonData = {
  id: 42,
  status: "OK",
  data: [867, 5309]
}
let { id, status, data: number } = jsonData;
console.log(id, status, number)
// 42, OK, [867, 5309]
  • 遍历Map结构
var map = new  Map();
map.set('first', 'hello');
map.set('second', 'world');
// 只获取键名
for(let [key] of map) {
    // ...
}
// 获取值
for(let [, value]) {
    // ...
}
  • 输入模块的指定方法
const {SourceMapConsumer, SourceNode} = require('source-map');
